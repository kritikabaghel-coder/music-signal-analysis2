SIGNAL PROCESSING CONCEPTS
═══════════════════════════════════════════════════════════════════════════════

1. ONSET STRENGTH
   └─ Measures energy in specific frequency bands
   └─ Detects percussive sounds (drum hits, attacks)
   └─ Uses STFT + mel-scale features

2. BEAT TRACKING
   └─ Aligns onsets to regular temporal grid
   └─ Uses Dynamic Time Warping (DTW)
   └─ Enforces constant tempo constraint

3. TEMPO ESTIMATION
   └─ Finds dominant periodicity via autocorrelation
   └─ Converts period to BPM: BPM = 60 / period_seconds

4. FRAME CONVERSION
   └─ Beat frames (audio indices) → Time (seconds)
   └─ Formula: time = frame_idx × hop_length / sr

ADVANTAGES & LIMITATIONS
═══════════════════════════════════════════════════════════════════════════════

Advantages:
  ✓ No machine learning needed
  ✓ Fully interpretable results
  ✓ Fast computation (2-5 min for 1000 files)
  ✓ Robust to audio quality variations
  ✓ Built on well-established theory (DTW, spectral analysis)

Limitations:
  ✗ May detect 2x or 0.5x actual tempo (double/half)
  ✗ Struggles with swing/shuffle rhythms
  ✗ Sensitive to silence/pauses
  ✗ Variable tempo (live recordings) problematic
  ✗ Polyrhythms detected as single dominant beat

Mitigation:
  • Use beat_regularity() to identify issues
  • Manual inspection of outliers
  • Check against music databases
  • Cross-validate with other beat trackers
