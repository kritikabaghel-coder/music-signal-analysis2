3. EXAMPLE WORKFLOW
═══════════════════════════════════════════════════════════════════════════════

Input:
   Audio file (e.g., "song.mp3" - 3 MB)

Processing:
   • Loads first 5 seconds only
   • Generates waveform plot (2-3 seconds)
   • Generates spectrogram plot (1-2 seconds)
   • Detects BPM (1-2 seconds)
   • Total: ~5 seconds

Output:
   • 🌊 Waveform visualization (time domain)
   • 🎼 Spectrogram visualization (frequency domain with colors)
   • ⏱️ BPM value (e.g., "95.2 BPM" → "Moderate (Blues, Rock)")
   • 📥 Download buttons for PNG images

═══════════════════════════════════════════════════════════════════════════════
5. WHAT YOU GET
═══════════════════════════════════════════════════════════════════════════════

🌊 Waveform
   • Time-domain amplitude plot
   • Useful for seeing audio shape, duration
   • Blue line = amplitude over time

🎼 Spectrogram
   • Frequency content over time
   • Color = loudness (bright = loud, dark = quiet)
   • X-axis: Time, Y-axis: Frequency (log scale)

⏱️ BPM (Beats Per Minute)
   • Detected tempo of the audio
   • Category interpretation:
     - Slow (60-80 BPM): Ballads, classical
     - Moderate (80-100 BPM): Blues, rock
     - Fast (100-130 BPM): Pop, disco, hip-hop
     - Very Fast (130+ BPM): Metal, electronic

7. TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

❌ App won't start
   Fix: pip install -r requirements_deployment.txt
   Then: streamlit run streamlit_app_light.py

❌ File upload fails
   Try: Different audio file or format
   Check: File size <5 MB
   Verify: File is not corrupted

❌ Processing takes >10 seconds
   Normal for first run (librosa initialization)
   Subsequent runs will be faster (cached)

❌ Plots not showing
   Fix: Clear browser cache
   Or: Hard refresh (Ctrl+Shift+R or Cmd+Shift+R)

❌ BPM is "N/A"
   Some audio (ambient, speech) may fail BPM detection
   Try: Different audio with clearer beats

❌ "ModuleNotFoundError"
   Fix: pip install streamlit librosa matplotlib numpy scipy
