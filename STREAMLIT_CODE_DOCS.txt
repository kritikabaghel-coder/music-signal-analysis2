═════════════════════════════════════════════════════════════════════════════
SECTION 1: FEATURE EXTRACTION
═════════════════════════════════════════════════════════════════════════════

CLASS: AudioFeatureExtractor
Purpose: Extract audio features using EXACT same pipeline as training (Step 2)

METHODS:

1. __init__(sr=22050)
   - Initializes with sample rate (22050 Hz default)
   - hop_length=512 (STFT frame hop)

2. compute_fft(y) → dict
   - FFT magnitude spectrum
   - Returns 6 features:
     * fft_mean, fft_std, fft_max, fft_min
     * fft_median, fft_sum

3. compute_stft_envelope(y) → dict
   - STFT magnitude and energy envelope
   - Returns 6 features:
     * stft_mean, stft_std, stft_max
     * stft_energy
     * envelope_mean, envelope_std

4. compute_mel_spectrogram(y) → dict
   - Mel-scale spectrogram
   - Returns 17 features:
     * mel_mean, mel_std, mel_max, mel_min
     * mel_band_0 through mel_band_12 (averages)

5. compute_mfcc(y) → dict
   - 13 Mel Frequency Cepstral Coefficients
   - Returns 26 features:
     * mfcc_0_mean, mfcc_0_std, ..., mfcc_12_std

6. compute_spectral_centroid(y) → dict
   - Spectral centroid (center of mass of spectrum)
   - Returns 4 features:
     * spectral_centroid_mean, _std, _max, _min

7. compute_spectral_bandwidth(y) → dict
   - Spectral spread around centroid
   - Returns 4 features:
     * spectral_bandwidth_mean, _std, _max, _min

8. compute_zero_crossing_rate(y) → dict
   - Zero crossings per frame
   - Returns 4 features:
     * zcr_mean, zcr_std, zcr_max, zcr_min

9. compute_chroma_features(y) → dict
   - 12 pitch classes (chromagram)
   - Returns 24 features:
     * chroma_0_mean, chroma_0_std, ..., chroma_11_std

10. extract_features(y) → dict
    - Orchestrates all feature extraction methods
    - Aggregates into single dictionary
    - Returns: ~86 total features (exact match to training)
    - Handles exceptions gracefully

TOTAL FEATURES: ~86 (matches training dataset)

═════════════════════════════════════════════════════════════════════════════
SECTION 2: AUDIO PROCESSING
═════════════════════════════════════════════════════════════════════════════

FUNCTION: load_audio_file(file_path, sr=22050) → (y, sr)
   - Loads audio using librosa.load()
   - Resamples to specified sample rate
   - Returns: audio array (y), sample rate (sr)
   - Error handling: Catches exceptions, logs error, returns (None, None)

FUNCTION: extract_features_from_file(file_path, sr=22050) → (features, y, sr)
   - Loads audio file
   - Creates AudioFeatureExtractor instance
   - Extracts features
   - Returns: (features_dict, y_array, sr)
   - Handles errors at each step

FUNCTION: get_genre_prediction(features_dict, model, encoder) → (genre, confidence)
   - Converts features dict to numpy array
   - Calls model.predict() for genre class
   - Calls model.predict_proba() for confidence
   - Decodes genre using label_encoder
   - Returns: (genre_string, confidence_float)
   - Error handling with logging

FUNCTION: detect_bpm(y, sr) → float (BPM)
   - Uses librosa.beat.beat_track()
   - Estimates tempo (beats per minute)
   - Returns: BPM value (float)
   - Returns None on error

FUNCTION: find_similar_songs(features_dict, features_df, top_n=5) → list[dict]
   - Computes cosine similarity with all training songs
   - Similarity = 1 - cos_distance
   - Selects top N most similar
   - Returns list of dicts:
     * Each dict has: file_path, genre, similarity
   - Handles missing CSV columns gracefully

KEY DESIGN DECISIONS
═════════════════════════════════════════════════════════════════════════════

1. Feature Extraction: EXACT same as training (Step 2)
   - Ensures model receives expected feature format
   - Maintains feature order consistency
   - Prevents mismatch between training and inference

2. Session State: Stores extracted data for tab switching
   - Avoids re-extraction on each tab click
   - Improves UX with responsive tabs

3. Model Caching: Uses @st.cache_resource
   - Avoids reloading models on every interaction
   - Significantly speeds up app

4. Temporary Files: Uses tempfile module
   - No permanent storage of uploads
   - Auto-cleanup after processing
   - Works cross-platform (Windows/Mac/Linux)

5. Error Handling: Graceful degradation
   - Individual feature failures don't crash app
   - Logged for debugging
   - User-friendly error messages

6. Modular Functions: ~10 functions + 1 class
   - Easy to test and maintain
   - Clear separation of concerns
   - Reusable components
