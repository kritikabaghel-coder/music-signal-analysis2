   3. Connect GitHub repo
   4. Select streamlit_app_light.py as main file
   5. Deploy!
   
   Settings:
   • Python version: 3.8+
   • timeout (client): 300 seconds (default)
   • maxUploadSize: 200 (MB)


Heroku Deployment:
   
   1. Create Procfile:
      web: streamlit run streamlit_app_light.py --logger.level=error
   
   2. Create runtime.txt:
      python-3.9.16
   
   3. Create .streamlit/config.toml:
      [logger]
      level = "error"
      
      [client]
      showErrorDetails = false
   
   4. Deploy:
      git push heroku main

5. CORE FUNCTIONS
═══════════════════════════════════════════════════════════════════════════════

load_audio_file(file_path, sr, max_duration)
   Purpose: Load audio with safety truncation
   Input: File path, sample rate, max duration
   Output: (audio_data, sr) tuple or (None, None)
   Safety: Automatic duration limit to prevent memory issues

detect_bpm(y, sr)
   Purpose: Extract tempo from audio
   Input: Audio array, sample rate
   Output: BPM float value or "N/A"
   Method: Librosa beat tracking with onset detection
   Error handling: Returns "N/A" on exception (graceful)

generate_waveform_plot(y, sr)
   Purpose: Create time-domain visualization
   Input: Audio array, sample rate
   Output: Matplotlib figure object
   Features: Time axis, amplitude scale, metadata box, grid

compute_spectrogram(y, sr, n_fft, hop_length)
   Purpose: Calculate STFT and convert to dB
   Input: Audio, sample rate, FFT parameters
   Output: 2D dB magnitude array
   Process: STFT → magnitude → amplitude_to_db conversion

generate_spectrogram_plot(y, sr, spectrogram_db)
   Purpose: Create frequency-domain visualization
   Input: Audio, sample rate, computed spectrogram
   Output: Matplotlib figure object
   Features: Colormap, colorbar, log frequency scale, metadata

process_audio_file(uploaded_file)
   Purpose: Complete pipeline for uploaded file
   Input: Streamlit uploaded_file object
   Output: Results dictionary or None
   Pipeline: Size check → Load → Spectrogram → BPM → Plots
   Error handling: Try-except at all stages, st.error() display

8. TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: "ModuleNotFoundError: No module named 'streamlit'"
   Solution: pip install -r requirements_deployment.txt
   Root cause: Dependencies not installed

Problem: App crashes on file upload
   Solution: Check file format, size, and integrity
   Debug: Try different audio file, check console logs
   Common cause: Corrupted audio file

Problem: Slow processing (>10 seconds)
   Solution: Normal for first run (librosa initialization)
   Optimization: Reduce MAX_DURATION or FIGSIZE
   Cloud: May be due to instance resources

Problem: "SSLError" on Streamlit Cloud
   Solution: Update SSL certificates
   Or: Use different domain/proxy
   Root cause: Network/security configuration

Problem: Plots not displaying
   Solution: Clear browser cache, try incognito window
   Alternative: Restart streamlit server
   Debug: Check console for matplotlib errors

Problem: BPM returns "N/A" for all files
   Solution: Ensure audio file has clear rhythm/beats
   Try: Different audio files with more obvious beats
   Note: Some ambient/spoken audio may fail BPM detection

Problem: Memory error on deployment
   Solution: Reduce MAX_DURATION to 3 seconds
   Or: Use smaller FIGSIZE
   Or: Upgrade instance size
   Root cause: Free tier resources exceeded

Problem: File upload takes forever
   Solution: Reduce MAX_FILE_SIZE limit
   Or: Use better network connection
   Cloud: May need faster instance
