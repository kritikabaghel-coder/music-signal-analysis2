8. LIBROSA BPM DETECTION SPECIFICATIONS
═══════════════════════════════════════════════════════════════════════════════

Function: librosa.beat.beat_track(y, sr)
   Inputs:
      • y (np.ndarray): Audio time series
      • sr (int): Sample rate in Hz
   Outputs:
      • tempo (float): Tempo (BPM)
      • beats (np.ndarray): Beat frame indices

How It Works:
   1. Onset Detection: Computes STFT and detects energy peaks
   2. Prior: Tempo prior encourages local consistency
   3. DP Search: Dynamic programming finds optimal beat sequence
   4. Unwrapping: Handles frequency ambiguities
   5. Conversion: Frame numbers converted to BPM using sample rate

Reliability:
   • Works well for music with clear rhythmic structure
   • May return "noisy" estimates for ambient/spoken content
   • Robust to audio quality variations
   • Industry standard for music analysis

Default Parameters (used):
   units='frames' (internal calculation)
   Implicit: hop_length based on FFT window
            tg=0.5 (tempo prior)
            prior_set='default'

11. TROUBLESHOOTING REFERENCE
═══════════════════════════════════════════════════════════════════════════════

Issue 1: "FileNotFoundError: data/dataset_metadata.csv"
   Root Cause: Step 1 (Dataset Loading) not run or file moved
   Solution: Run: python dataset_loader.py
   Prevention: Keep project structure intact

Issue 2: No files processed (0 results)
   Root Cause: genre column missing/empty in CSV
   Solution: 
      1. Open data/dataset_metadata.csv
      2. Check 'genre' column exists
      3. Verify data is populated
   Prevention: See Step 1 dataset loader requirements

Issue 3: All BPM values are "N/A"
   Root Cause: Audio files missing or corrupted
   Solution:
      1. Check data/genres/ folder structure
      2. Verify audio files exist
      3. Try manual load: librosa.load(filepath)
   Prevention: Verify dataset before running

Issue 4: Script runs very slowly
   Root Cause: Beat tracking is computationally intensive
   Solution: This is normal (~30-50 sec for 10 files)
   Optimization: Reduce MAX_DURATION in script

Issue 5: Memory error / Out of memory
   Root Cause: System running low on RAM
   Solution:
      1. Close other applications
      2. Reduce MAX_DURATION in line 46
   Prevention: Ensure 4+ GB RAM available

Issue 6: ImportError for librosa/pandas/numpy
   Root Cause: Packages not installed
   Solution: pip install -r requirements.txt
   Prevention: Always install requirements first
