2. SCRIPT FUNCTIONALITY
══════════════════════

The detect_bpm.py script provides the following functions:

load_audio_file(file_path, sr=22050, max_duration=10)
   Purpose: Load audio with automatic safety truncation
   Inputs:
      - file_path: Path to audio file (WAV, MP3, etc.)
      - sr: Sample rate for resampling (default 22050 Hz)
      - max_duration: Maximum duration in seconds (default 10s)
   Output: (audio_data, sample_rate) tuple or (None, None) on error
   Behavior:
      • Loads audio using librosa.load()
      • Automatically truncates to max_duration if longer
      • Returns error-safe None values on failure
      • Logs all operations (load success, truncation, errors)

detect_bpm(y, sr)
   Purpose: Detect BPM using beat tracking algorithm
   Inputs:
      - y: Audio time series data (numpy array)
      - sr: Sample rate (Hz)
   Output: BPM value (float) or "N/A" (string) on error
   Behavior:
      • Uses librosa.beat.beat_track(y, sr)
      • Converts result to float for consistency
      • Returns "N/A" on any exception (maintains stability)
      • Logs detected BPM and any errors

extract_bpm_for_file(file_path, genre, sr=22050, max_duration=10)
   Purpose: Complete BPM extraction for single audio file
   Inputs:
      - file_path: Path to audio file
      - genre: Genre label string
      - sr: Sample rate (Hz)
      - max_duration: Maximum duration (seconds)
   Output: Dictionary with keys: file_name, genre, bpm
   Behavior:
      • Calls load_audio_file() with error handling
      • Calls detect_bpm() on loaded audio
      • Returns structured result dict
      • Logs file processing status
      • Handles failures gracefully (returns genre + "N/A" BPM)

process_sample_per_genre(metadata_file, sr=22050, max_duration=10)
   Purpose: Main orchestration - process one file per genre
   Inputs:
      - metadata_file: Path to dataset_metadata.csv (from Step 1)
      - sr: Sample rate (Hz)
      - max_duration: Maximum duration (seconds)
   Output: Statistics dictionary with keys:
      - total_genres: Number of unique genres
      - successful: Count of successful BPM detections
      - failed: Count of failed detections
      - results: List of result dictionaries
      - output_file: Path to output CSV file
   Behavior:
      • Reads dataset_metadata.csv
      • Groups files by genre
      • Selects first file per genre (10 files for GTZAN)
      • Extracts BPM from each file
      • Logs summary statistics
      • Returns structured stats dict

save_results_to_csv(results, output_file)
   Purpose: Export results to CSV format
   Inputs:
      - results: List of result dictionaries
      - output_file: Path where CSV should be saved
   Output: Boolean (True if successful, False on error)
   Behavior:
      • Converts results list to pandas DataFrame
      • Saves as CSV with headers
      • Logs output file path
      • Returns success status

print_summary(stats)
   Purpose: Display formatted console output
   Inputs:
      - stats: Statistics dictionary from process_sample_per_genre()
   Output: None (prints to console)
   Behavior:
      • Displays header and separator lines
      • Shows genre processing statistics
      • Lists BPM results for each genre
      • Calculates and displays BPM statistics (mean, min, max, std dev)
      • Uses box-drawing characters and emojis for readability

4. SPECIFYING BPM DETECTION DETAILS
════════════════════════════════════

BPM Detection Algorithm
   - Library: librosa 0.10+
   - Function: librosa.beat.beat_track(y, sr)
   - Method: Onset-based beat tracking with constant-deceleration assumption
   - Onset Detection: Energy-based (uses STFT)
   - Dynamic Programming: Finds optimal beat positions
   - Output: Single tempo value (BPM) + beat frames
   
   Why Librosa?
      • Industry standard for music information retrieval (MIR)
      • Robust to audio variations
      • Handles silence and noise well
      • Fast C-backend implementation

Audio Loading Parameters
   Sample Rate: 22050 Hz
      • Standard for music analysis (half of CD 44.1 kHz)
      • Captures up to 11025 Hz (above human hearing ~20 Hz)
      • Reduces computation vs higher rates
      • Industry standard in MIR research
   
   Max Duration: 10 seconds
      • Prevents memory exhaustion with very long files
      • BPM stabilizes within first 10s of audio
      • Speeds up processing significantly
      • Can change in code if needed (line 46)
      • Automatic truncation if audio exceeds limit

Error Handling Strategy
   "N/A" Value Storage:
      • Graceful degradation if BPM detection fails
      • Never crashes - always returns a result
      • Preserves file name and genre information
      • Enables downstream filtering/analysis
   
   Failure Scenarios:
      • Audio file corrupted → logs ERROR + returns N/A
      • Sample rate mismatch → librosa handles automatically
      • Too-short audio → beat tracking may return N/A
      • Out of memory → caught by exception handler
      • Invalid file path → logger catches immediately


5. PROCESSING OVERVIEW
══════════════════════

Execution Pipeline (per genre):
   1. Read dataset_metadata.csv
   2. Group rows by 'genre' field
   3. Select first row of each genre group (10 times for 10 genres)
   4. For each file:
      a. Call load_audio_file() → get (y, sr) or (None, None)
      b. Call detect_bpm(y, sr) → get bpm value or "N/A"
      c. Create result dict {file_name, genre, bpm}
      d. Log: "✓ {genre}/{filename}: {bpm} BPM"
   5. Build results list (10 items)
   6. Convert to pandas DataFrame
   7. Save as CSV to outputs/bpm_results.csv
   8. Calculate statistics (mean, min, max, std dev)
   9. Print summary to console
   10. Log completion with stats

Timing Expectations
   • Loading audio: ~0.5-1 second per file
   • Beat tracking: ~1-3 seconds per file (depends on audio complexity)
   • Total runtime: ~15-40 seconds for 10 files
   • CSV writing: ~100 ms
   • Total: ~30-50 seconds typically

Memory Usage
   • Per audio file: ~1-2 MB (10s @ 22050 Hz)
   • STFT/processing: ~5-10 MB per file
   • Peak usage: ~20-30 MB total (all files + overhead)
   • Safe for any modern system

9. TROUBLESHOOTING
═══════════════════

Problem: "FileNotFoundError: data/dataset_metadata.csv not found"
   Cause: metadata file missing (Step 1 not completed)
   Solution:
      1. Run dataset loader first: python dataset_loader.py
      2. Or check data/ folder contains dataset_metadata.csv

Problem: No files processed / 0 results
   Cause: Metadata file exists but genre column missing/empty
   Solution:
      1. Open data/dataset_metadata.csv in spreadsheet
      2. Verify 'genre' column exists with valid values
      3. Verify 'file_path' column points to actual files

Problem: "N/A" BPM for all files
   Cause: Audio files missing or beat tracking failing
   Solution:
      1. Verify data/genres/ folder contains audio files
      2. Check logs/bpm_detection.log for specific errors
      3. Try manual test: librosa.load(filepath); librosa.beat.beat_track(y, sr)

Problem: Script runs very slowly
   Cause: Beat tracking algorithm is computationally intensive
   Solution:
      1. Reduce MAX_DURATION in script (line 46)
      2. Note: This is normal - beat tracking is expensive
      3. Typical time: 30-50 seconds for 10 files

Problem: Out of memory error
   Cause: Files are very long or max_duration too high
   Solution:
      1. Reduce MAX_DURATION (line 46): MAX_DURATION = 5
      2. Process fewer genres at a time
      3. Close other applications to free RAM

Problem: ImportError: librosa not found
   Cause: Librosa library not installed
   Solution:
      pip install librosa
      Or: pip install -r requirements.txt

Problem: Permission denied writing to outputs/ or logs/
   Cause: Missing folder or read-only permissions
   Solution:
      1. Verify outputs/ and logs/ folders exist
      2. Create if needed: mkdir outputs logs
      3. Check folder permissions (right-click → Properties)
      4. On Linux/Mac: chmod 755 outputs logs

13. ADVANCED FFT PARAMETER EXPLANATION
════════════════════════════════════════

Why These Settings?

Beat Tracking Algorithm Details:
   1. Onset Detection:
      • Librosa detects energy peaks in STFT
      • Identifies Note/beat onsets
      • Uses frames to identify percussive events
   
   2. Dynamic Programming:
      • Finds optimal sequence of beat frames
      • Assumes constant tempo + small deviations
      • Balances: staying on beat vs. following onsets
   
   3. Tempo Estimation:
      • Converts frame delays to BPM
      • Uses sample rate: BPM = (60 * sr) / frames_between_beats
      • Typical range: 40-240 BPM (musical range)

Audio Processing Details:
   Sample Rate 22050 Hz:
      • Standard Nyquist: 11025 Hz max frequency
      • Human hear range: 20 Hz - 20 kHz
      • 22050 Hz handles music well (loses ultrasonic content)
      • Used by most MIR researchers/librosa docs
   
   Max Duration 10s:
      • BPM analysis often stabilizes in 5-10s
      • First verse usually representative of song tempo
      • Balances accuracy vs. performance
      • Can be extended if needed
