CORE REQUIREMENT: Machine Learning Classification

  ✓ Train a Genre Classification Model
    ├─ Algorithm: RandomForestClassifier (no deep learning)
    ├─ Framework: scikit-learn (not TensorFlow/PyTorch)
    ├─ Purpose: Classify music by genre
    └─ Input: 80+ spectral features from Step 2

  ✓ Prepare data: Separate features (X) and labels (y)
    ├─ Features: 82 digital features per audio file
    ├─ Labels: 10 genre classes
    ├─ Encoding: LabelEncoder for reproducibility
    └─ Output: X (1000×82), y (1000,)

  ✓ Encode genre labels
    ├─ blues=0, classical=1, country=2, ..., reggae=9
    ├─ Reproducible: same order every time
    └─ Used: LabelEncoder from sklearn

  ✓ Split dataset into train and test sets (80-20 split)
    ├─ Training: 800 samples (80%)
    ├─ Testing: 200 samples (20%)
    ├─ Stratified: maintains class distribution
    └─ Function: train_test_split(stratify=y)

  ✓ Train model: Use RandomForestClassifier
    ├─ Parameters: n_estimators=100, random_state=42
    ├─ Training: model.fit(X_train, y_train)
    └─ Fitness: evaluated on both train and test sets

  ✓ Evaluate model: Print multiple metrics
    ├─ Accuracy (overall %): typically ~75-80%
    ├─ Classification Report (per-class metrics):
    │  - Precision: what % of predicted X are actually correct?
    │  - Recall: what % of actual X did we correctly find?
    │  - F1-Score: harmonic mean of precision and recall
    │  - Support: number of samples per class
    │
    ├─ Confusion Matrix (actual vs predicted):
    │  - Diagonal = correct predictions
    │  - Off-diagonal = misclassifications
    │  - Shows which genres are confused with which
    │
    └─ Per-Class Accuracy (test set)
       - Individual accuracy for each of 10 genres
       - Identifies easy vs hard-to-classify genres

  ✓ Feature Importance Analysis
    ├─ Method: Mean Decrease in Impurity (MDI)
    ├─ Importance: how much each feature contributes to splits
    ├─ Output: Top 20 features ranked by importance
    └─ Typical top features: MFCCs, spectral centroid, spectral features

  ✓ Save trained model: pickle or joblib format
    ├─ Format: joblib (preferred, more efficient)
    ├─ File: trained_model.pkl
    ├─ Contents: RandomForestClassifier + LabelEncoder + metadata
    ├─ Usage: Load and predict on new audio files
    └─ Function: joblib.dump(model, filepath)

  ✓ Code Quality Constraints
    ├─ ✓ Modular: GenreClassificationModel, ModelEvaluator, ClassificationPipeline
    ├─ ✓ Clean: Clear naming, docstrings, type hints
    ├─ ✓ Object-oriented: Class-based architecture
    ├─ ✓ Reproducibility: random_state=42 throughout
    │  - GenreClassificationModel(random_state=42)
    │  - train_test_split(..., random_state=42)
    │  - RandomForestClassifier(..., random_state=42)
    │
    └─ ✓ No deep learning: sklearn only, no TensorFlow/PyTorch/Keras
